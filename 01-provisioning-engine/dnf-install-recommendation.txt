## What Each Extension Does

### Performance & Monitoring
| Extension | What It Does |
|-----------|-------------|
| `edb_wait_states` | Captures wait events, session states, query performance data |
| `pg_stat_statements` | Tracks execution stats of all SQL statements |
| `auto_explain` | Automatically logs execution plans for slow queries |
| `pg_buffercache` | Inspect shared buffer cache contents |
| `pg_freespacemap` | Inspect free space map |
| `indexadvisor` | Recommends indexes to improve query performance |
| `sqlprofiler` | Profiles SQL execution |

### Security
| Extension | What It Does |
|-----------|-------------|
| `pgaudit` | Detailed audit logging of database activities |
| `sqlprotect` | Protects against SQL injection attacks |
| `sslutils` | Manage SSL certificates within the database |

### Maintenance & Automation
| Extension | What It Does |
|-----------|-------------|
| `pg_cron` | Schedule and run periodic jobs inside the database |
| `pg_repack` | Reorganize tables online without locking (like `VACUUM FULL` but online) |

### High Availability
| Extension/Tool | What It Does |
|----------------|-------------|
| `efm` | Automatic failover manager |
| `pgpool` | Connection pooling, load balancing, failover |

### Reporting & Diagnostics
| Tool | What It Does |
|------|-------------|
| `edb-pwr` | Generates performance workload reports from `edb_wait_states` and `pg_stat_statements` data |
| `edb-lasso` | Collects diagnostic data for EDB support cases |

## Post-Installation Configuration

### 1. Initialize and Start EPAS 17

```bash
# Initialize the database cluster (if not already done)
sudo /usr/edb/as17/bin/edb-as-17-setup initdb

# Enable and start the service
sudo systemctl enable edb-as-17
sudo systemctl start edb-as-17
```

### 2. Configure `shared_preload_libraries`

Edit `/var/lib/edb/as17/data/postgresql.conf`:

```ini
shared_preload_libraries = 'edb_wait_states, pg_stat_statements, pgaudit, pg_cron, auto_explain, pg_repack'
```

### 3. Configure Extension-Specific Parameters

```ini
# pg_stat_statements
pg_stat_statements.max = 10000
pg_stat_statements.track = all
pg_stat_statements.track_planning = on

# pgaudit
pgaudit.log = 'write, ddl'
pgaudit.log_parameter = on
pgaudit.log_relation = on

# auto_explain
auto_explain.log_min_duration = '5s'
auto_explain.log_analyze = on
auto_explain.log_buffers = on
auto_explain.log_timing = on
auto_explain.log_nested_statements = on

# pg_cron
cron.database_name = 'edb'

# edb_wait_states
edb_wait_states.sampling_interval = 1
edb_wait_states.retention_period = 604800
edb_wait_states.enable_collection = true
```

### 4. Restart EPAS

```bash
sudo systemctl restart edb-as-17
```

### 5. Create Extensions in Database

```sql
\c edb

CREATE EXTENSION IF NOT EXISTS pg_stat_statements;
CREATE EXTENSION IF NOT EXISTS pgaudit;
CREATE EXTENSION IF NOT EXISTS pg_cron;
CREATE EXTENSION IF NOT EXISTS pg_repack;
CREATE EXTENSION IF NOT EXISTS pg_buffercache;
CREATE EXTENSION IF NOT EXISTS pg_freespacemap;
CREATE EXTENSION IF NOT EXISTS sslutils;
CREATE EXTENSION IF NOT EXISTS indexadvisor;
CREATE EXTENSION IF NOT EXISTS sqlprofiler;
CREATE EXTENSION IF NOT EXISTS sqlprotect;
```

> **Note:** `auto_explain` and `edb_wait_states` do **not** require `CREATE EXTENSION` — they work at the server level via `shared_preload_libraries`.

## Quick Verification Script

```bash
#!/bin/bash

echo "=== EPAS 17 Service Status ==="
sudo systemctl status edb-as-17 --no-pager

echo ""
echo "=== Installed Packages ==="
rpm -qa | grep -E 'edb-as17|edb-pwr|edb-lasso' | sort

echo ""
echo "=== Shared Preload Libraries ==="
sudo -u enterprisedb psql -d edb -c "SHOW shared_preload_libraries;"

echo ""
echo "=== Installed Extensions ==="
sudo -u enterprisedb psql -d edb -c "\dx"

echo ""
echo "=== pg_stat_statements Status ==="
sudo -u enterprisedb psql -d edb -c "SELECT count(*) FROM pg_stat_statements;"

echo ""
echo "=== edb_wait_states Status ==="
sudo -u enterprisedb psql -d edb -c "SELECT count(*) FROM edb_wait_states_data();"

echo ""
echo "=== pg_cron Jobs ==="
sudo -u enterprisedb psql -d edb -c "SELECT * FROM cron.job;"

echo ""
echo "=== pgaudit Settings ==="
sudo -u enterprisedb psql -d edb -c "SHOW pgaudit.log;"
```
